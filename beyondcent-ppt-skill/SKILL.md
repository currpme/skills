# 博云风格 PPT 制作技能

## 目标

在 `博云ppt模版.pptx` 的视觉语言基础上，生成风格统一、结构清晰、可直接汇报的 PPT。

这个技能不是通用美化，而是严格复用模板风格：
- 保持博云模板的企业汇报气质
- 保持蓝白主视觉、浅色科技装饰、留白方式和标题层级
- 保持简洁、规整、偏正式的表达方式
- 保持“图文并重、以图佐证、以一句话结论带出正文”的呈现逻辑

素材位于本 skill 目录下的 `assets/`

## 二、视觉规则

### 1. 画布与背景
- 使用 16:9 宽屏。
- 页面整体以白底为主。
- 背景装饰只做浅灰/浅蓝科技感点缀，不能喧宾夺主。

### 2. 品牌与色彩
- 主标题蓝：`#1A2CB3`
- 辅助蓝：`#0DA3D5`
- 可替代浅蓝：`#00B0F0`
- 正文深色：`#1F2329`
- 说明灰：`#7F7F7F`
- 背景白：`#FFFFFF`

使用规则：
- 标题、重点词、结构标识优先用主标题蓝
- 辅助图形、节点、目录连接点可用辅助蓝
- 正文默认用深灰/黑色
- 不引入新的高饱和品牌色，除非内容中的原始截图本身带色

### 3. 字体
- 正文以微软雅黑为主
- 英文和数字可兼容 Arial / Inter 风格，但可编辑文本仍优先微软雅黑

推荐层级：
- 封面主标题：36pt
- 目录标题：32pt
- 正文页标题：24pt，加粗，蓝色
- 二级小标题/强调语：14-16pt
- 正文段落：12pt
- 表格正文：10.5-12pt
- 页脚/注释：9-11pt

字体要求：
- 不混用花哨字体
- 不随意给正文加粗
- 重点只对关键数字、关键词、结论短语加粗或着色
- 一页内字号层级不超过 4 档

## 三、版式规则

### 1. 安全边距
- 左边距约 `0.35 inch`
- 右边距约 `0.35-0.5 inch`
- 顶部标题区域高度约 `0.7-1.0 inch`
- 正文通常从页面上方 `1.0-1.3 inch` 后开始
- 底部预留页码或装饰安全区

### 2. 对齐逻辑
- 标题统一左对齐
- 正文、图片、表格尽量沿同一左边线展开
- 多个模块优先做上对齐或左对齐
- 同层级对象间距保持一致

### 3. 留白要求
- 每页只保留一个核心结论
- 一页最多 2 个主要视觉重心
- 文字与图片之间要有明确留白
- 图表和截图不要贴边，不要拥挤叠放

### 4. 线条与边框
- 线条以细线为主，偏浅蓝或浅灰
- 不用厚重描边
- 不使用强阴影、发光、3D、渐变特效

## 四、页面类型

### 1. 封面页
封面页使用方式：
- 大标题居中或偏中上
- 蓝色主标题
- 白底
- 左上保留 `assets/master-top-left-decor.png`
- 右上保留 `assets/brand-logo.png`
- 右下或中下优先使用 `assets/cover-visual-right-a.png` 或 `assets/cover-visual-right-b.png`
- 左侧大背景优先使用 `assets/cover-bg-left-a.png` 或 `assets/cover-bg-left-b.png`

封面文案建议结构：
- 主标题
- 副标题（可选）
- 日期 / 汇报部门 / 汇报人（如需要）

### 2. 目录页
特征：
- 左侧为“目录”大字标题
- 中间有纵向连接线
- 每个章节通过圆点 + `Part 01 / Part 02 / Part 03` 表示
- 右侧放章节名称

要求：
- 章节数优先控制在 3-5 个
- 超过 5 个时拆成两页目录
- `Part XX` 格式统一
- 章节名尽量为名词短语

### 3. 章节页 / 过渡页
特征：
- 左上保留蓝色标题
- 可搭配一张大图、结构图或系统截图
- 适合做章节开场或重点引入

页面内容：
- 业务集群
- 推理任务
- 平台能力
- 架构规划
- 模型能力

### 4. 正文页
结构建议：
- 顶部：页面标题
- 中上部：一句话总结 / 引导语
- 中部：正文、截图、表格、流程图、说明文字
- 底部：补充说明或页码

规则：
- 标题下方到正文之间要有明显间距
- 一段正文尽量控制在 3-6 行
- 列表不要过密，优先拆解为短句
- 支持“左文右图”或“上文下图”

### 5. 表格页
要求：
- 表格居中摆放，周边保留足够留白
- 表头简洁，表头字体 11pt 左右
- 表格正文一般 10.5-12pt
- 边框细，颜色淡，不要黑粗边框
- 表格上方通常先有一段结论性文字或背景说明

### 6. 图片 / 截图页
要求：
- 截图清晰，不可拉伸变形
- 截图四周保留安全边距
- 同页如有两张截图，尺寸和对齐方式必须统一
- 图片下方可加简短说明，但不能长段解释
- 截图本身色彩复杂时，周边版式要保持克制

### 7. 结尾页
模板结尾页特征：
- 中间区域出现 `THANK YOU`
- 下方放联系电话、邮箱、二维码等联系信息
- 风格简洁，不要堆内容
- 保持 `assets/brand-logo.png` 的品牌体系一致

## 五、文案组织方式

### 1. 标题写法
标题应短、稳、专业，优先使用：
- 平台功能概览
- 整体架构设计
- 关键能力建设
- 当前建设成果
- 运行现状分析
- 验收结果说明
- 后续规划建议

避免：
- 口语化标题
- 情绪化标题
- 过长标题
- 一整个句子直接做标题

### 2. 正文写法
正文用结论先行的企业汇报表达：
- 先说现状 / 目标
- 再说措施 / 能力
- 最后说结果 / 价值

推荐句式：
- 当前平台已完成……
- 围绕……构建了……能力
- 通过……实现了……
- 在……场景下，支撑了……
- 后续将继续推进……

### 3. 信息密度
每页只回答一个问题：
- 这页讲什么
- 这页证明什么
- 这页给听众留下什么结论

不能出现：
- 一页塞满多个主题
- 一页既讲背景又讲架构又讲结果
- 大段无重点文字直接堆上去

## 六、图表与图形风格

### 1. 图表
- 使用简洁商务图表
- 颜色优先蓝色系
- 图例精简
- 坐标轴与网格线弱化
- 不用炫技型图表

### 2. 流程图 / 架构图
- 使用横向流程或分层结构优先
- 节点尽量统一圆角矩形样式
- 箭头细且清晰
- 主流程突出，辅助流程弱化
- 使用蓝色、浅蓝、灰色形成层级

### 3. 图标
- 图标风格保持扁平、统一
- 不混搭线性图标和拟物图标
- 不使用夸张彩色插画风元素

## 七、内容迁移原则

当需要把其它资料迁移进博云模板时，必须遵循以下原则：

### 1. 先重构，再排版
- 提炼标题
- 提炼一页一句话结论
- 拆分过长段落
- 判断哪些内容适合表格，哪些适合图示，哪些适合正文

### 2. 保留事实，不照搬旧版式
- 保留原始数据、结构、结论
- 不保留旧 PPT 的杂乱版式
- 全部按博云模板风格重新落版

### 3. 优先图文化表达
优先图形化的内容：
- 平台架构
- 流程关系
- 能力分层
- 指标对比
- 节点关系

### 4. 页面节奏建议
完整汇报建议按以下节奏组织：
- 封面
- 目录
- 背景 / 目标
- 现状 / 问题
- 架构 / 功能
- 运行情况 / 数据结果
- 规划 / 价值
- 总结 / 结束页

## 八、制作禁区

以下做法会破坏博云模板风格，必须避免：
- 使用大面积渐变背景
- 使用强阴影、发光、3D 效果
- 使用高饱和杂色
- 标题和正文使用不同风格字体
- 一页内堆太多截图
- 图片比例失真
- 表格线条过粗
- 页面元素不对齐
- 把装饰图形做得比正文还抢眼
- 大段文字原封不动贴入 PPT

## 九、标准制作流程

### 第一步：识别页面类型
判断当前页属于：
- 封面
- 目录
- 章节页
- 正文页
- 表格页
- 截图页
- 结尾页

### 第二步：提炼核心信息
每页先写出：
- 页面标题
- 一句话结论
- 3 个以内支撑点

### 第三步：选择版式
根据内容选择：
- 上文下图
- 左文右图
- 图表居中
- 大图引导
- 目录型结构

### 第四步：统一视觉
检查：
- 字体是否统一
- 颜色是否统一
- 边距是否统一
- 对齐是否统一
- 图片比例是否正确

### 第五步：汇报化润色
将内容改写为：
- 更简洁
- 更专业
- 更适合演示口播
- 更符合企业汇报语气

## 十、交付前检查清单

### 视觉检查
- 是否仍然是博云风格，而不是普通蓝色 PPT
- `assets/brand-logo.png` 是否统一放置在右上角
- 标题颜色、字号、位置是否一致
- 页面是否干净、规整、有留白

### 内容检查
- 每页是否只有一个核心主题
- 是否出现大段难读文字
- 是否存在可以图示却仍然堆文字的情况
- 标题是否准确概括页面内容

### 版式检查
- 元素是否对齐
- 间距是否一致
- 图片是否清晰
- 表格是否过密
- 是否有元素贴边或越界

### 汇报检查
- 页面顺序是否符合讲述逻辑
- 章节之间是否过渡自然
- 是否能支持现场讲解
- 是否具备客户可看、领导可看、内部可汇报的正式程度

## 十一、执行口令

当收到“参考博云模板生成 PPT”一类任务时，默认执行以下规则：

1. 先识别内容结构并拆页
2. 再按博云模板页面类型匹配版式
3. 严格使用蓝白商务科技风
4. 标题统一为蓝色左对齐样式
5. 正文优先 12pt 微软雅黑
6. 截图、表格、流程图优先规整落版
7. 不做花哨设计，只做企业级高完成度汇报页
8. 最终输出必须看起来像“博云原模板延展出来的同系列文档”

## 十二、适用任务示例

本技能适用于：
- 验收报告 PPT
- 平台能力介绍 PPT
- 项目汇报 PPT
- 技术方案汇报 PPT
- 运维平台介绍 PPT
- 模型平台介绍 PPT
- 产品规划 / 建设规划汇报 PPT

## 十三、最终要求

无论内容来自 Word、Markdown、旧 PPT、表格还是访谈纪要，最终生成结果都必须满足：

**内容是新的，但风格必须像博云原模板。**

也就是说：
- 版式像同一套模板
- 配色像同一品牌体系
- 字体层级像同一母版规则
- 页面节奏像同一位设计师延展出来的成稿
